## ✨ 功能特性

### 📢 动态推送
- 实时监控 UP 主发布的 **图文动态、视频投稿、专栏文章** 等。
- 支持 **转发动态**，自动显示原作者及原始内容。
- **智能折叠**：当图片数量超过阈值（默认 3 张）时，自动打包为 **合并转发消息**，防止刷屏。
- **置顶识别**：可识别并推送新增置顶动态，旧置顶不会重复推送。
- **过期过滤**：超过 `max_dynamic_age` 的旧动态不再推送（避免重启后推送积压消息）。
- **开奖过滤**：可自动丢弃开奖类动态。

### 🔴 直播通知
- **开播提醒**：包含封面、标题、跳转链接和开播时间。
- **下播提醒**：自动统计本次直播时长。

### 🛡️ 防风控机制
- 支持 **轮询抖动**，模拟真人查询行为，降低被风控概率。
- 内置 **凭证自动刷新**（每 6 小时检测一次）。

### 🛠️ 在线管理
- 支持通过群聊指令进行 **订阅增删查**，管理员限定。

## ⚙️ 配置说明

插件首次运行会自动生成配置模板，下面是 `config.toml` 关键字段说明：

```toml
[plugin]
enabled = true                # 插件总开关

[settings]
poll_interval   = 120         # 轮询基准间隔（秒），建议 ≥ 60
poll_jitter     = 10          # 轮询抖动秒数，实际间隔为 base ± jitter
admin_qqs       = ["114514"]  # 管理员 QQ 号列表
max_images      = 3           # 单条动态最大同消息图片数，超过则合并转发
ignore_lottery  = true        # 自动丢弃开奖类动态
max_dynamic_age = 3600        # 动态最大有效时长（秒），过老的不再推送

# B 站凭证（建议配置，可访问受限内容并提升稳定性）
# 获取方式：浏览器登录 B 站 → F12 → Application → Cookies → https://www.bilibili.com
[settings.credential]
sessdata  = "your_sessdata_here"
bili_jct  = "your_bili_jct_here"
buvid3    = "your_buvid3_here"

# 订阅列表
# - uid:    单个 UP 主 UID
# - uids:   多个 UID 列表（与 uid 二选一或并存）
# - groups: 推送目标群号列表
[subscriptions]
users = [
    { uid  = "114514",                groups = ["12345678", "87654321"] },
    { uids = ["114514", "1919810"],   groups = ["12345678", "87654321"] },
    { uid  = "36081646",              groups = ["12345678"] }
]
```

> 💡 如果觉得手动配置 `users` 麻烦，可以留空，进群后用 `/B动态 add` 指令进行订阅。

## 📋 指令列表

> ⚠️ 所有指令仅限 **管理员（admin_qqs）** 在 **群聊** 中使用。

| 指令 | 说明 |
| --- | --- |
| `/B动态 help` | 查看帮助信息 |
| `/B动态 start` | 启动监控 |
| `/B动态 stop` | 停止监控 |
| `/B动态 status` | 查看运行状态及订阅数 |
| `/B动态 add <UID>` | 在当前群订阅指定 UP 主 |
| `/B动态 remove <UID>` | 在当前群移除指定订阅（仅可移除指令添加的） |
| `/B动态 list` | 列出当前群所有订阅 |
| `/B动态 info <UID>` | 查询该 UP 主当前直播状态 |
| `/B动态 test <UID>` | 推送一条测试动态到当前群 |

## 📂 文件结构

```
plugin/
├── plugin.py            # 插件主代码
├── config.toml          # 配置文件
├── history.json         # [自动生成] 动态/直播历史记录，用于去重
└── subscriptions.json   # [自动生成] 指令添加的订阅信息
```

> ⚠️ `history.json` 与 `subscriptions.json` 由插件自动维护，**请勿手动修改**。

---

## ⚠️ 注意事项

1. **风控建议**：尽管已加入抖动机制，仍建议 `poll_interval ≥ 60`，避免 API 被封禁。
2. **凭证维护**：插件内置每 6 小时自动刷新凭证；若 Cookie 彻底失效，需手动更新配置。
3. **首次订阅**：新订阅 UID 时，第一次轮询会以当前最新动态作为基准，**不推送历史动态**。
4. **请勿手动修改** `history.json` 和 `subscriptions.json`，否则可能导致重复推送或订阅异常。
5. **固定订阅 vs 动态订阅**：
   - 配置文件中的订阅为"固定订阅"，**不可通过指令移除**。
   - 通过 `/B动态 add` 添加的为"动态订阅"，可通过 `/B动态 remove` 移除。
